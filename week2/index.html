Forensics Investigation

From this week’s video lecture we went over advanced forensics methods and tools that we can use to help us with forensics investigation. First we identify what scenarios or incidents would trigger a need for forensic investigation.

Incidents of Fraud, Theft, Hacker Intrusion/ Data breaches, inappropriate use of Internet, Child Exploitation, civil or criminal litigation…etc are all cases that would trigger an need for forensic investigation. After these incidents occurred and the need for forensic investigation is required, evidence needs to be gathered quickly since we know the data in computer can be lost or overwritten as time goes on, however, the important first step before anything is done is to log down the TIME.

Noting down the time the incidence occurred and the time when you performed any evidence acquisition will help you develop a timeline. This is very important. After the time is logged, then is the process of gathering the data. This can be classified into three broad methods.

  • Live forensics: while the computer is still on and live, gather data in running machine.
  • Post-mortem forensics: gather data in the memory after it is removed from machine.
  • Network based forensics: gather data from the network traffic, dealing with volatile and dynamic information.
Data that are normally not seen are collected and different tools can help acquire those data for analysis.

What are evidences and how are they collected?

Evidences in computer forensics are considered mainly the data portion of the hardware. Since all hardware can store data in the memory so they can be found in locations like:

  • In networks- firewalls, IDS, routers
  • operating system- computer ram, hard drives
  • database and applications- software that are used and retain data or logs
  • removable media- cd, usb, external drives

To collect data in form of evidences for forensic investigation, there are rules that must apply. 1. Minimize data loss if possible. 2. Record everything. 3. Analyze all data collected. 4. Report findings. If any part of the rule is not followed, then the evidence may not get accepted in court and the case may get dismissed.

collection process:

First mentioned, log the time and then we must always preserve the integrity of the evidence meaning that we should create a cryptographic hash of the disk. This can be seen like a unique signature and a copy of the original is created for analysis since we do not want to alter or contaminate the original copy so all analysis is done on the copy and the copy’s signature must match the original to preserve the integrity of the evidence. After the copy is obtained and verified, the original is locked away so they cannot be contaminated. If an evidence is contaminate, then let it stay contaminated because this will be considered as compromised evidence.

Order of collection

When collecting evidences, there is something called volatility to consider. Generally you would want to start from data that is more volatile to less volatile meaning data that can be lost quickly if we don’t get to it first. So the order goes in from more volatile to less volatile as follows:

  • System Memory: memory that is in RAM that will lost if system shutdown.
  • Temporary File Systems: ie. swap files and paging files.
  • Process Table & Network Connections: helpful process information that can be used shold be dumped
  • Network Routing Information & ARP Cache: This may help identify and match system that connect to it
  • Forensics Acquisition of Disks: Making image of the disk
  • Remote Logging & Monitoring Data: this helps seeing what is going on remotely
  • Physical configuration & network toplology: This help getting a whole view of connections
  • Backups: These are the less volatile and may contain good evidence

Where to collect?

Locations that needs data collections are:

  • Memory: both virtual and physical memory should be dumped, these may contain very useful information.(ie. keystrokes, processes, usernames and passwords...etc)
  • Drives: the whole physical hard disk should be imaged as well as the logical which is the partition of the drive should also be imaged as well.
  • Network traffic: data from packet captures should be dumped since this can be useful for comparison to some data that can be used to prove it come from a system.

Tools used:

Tools that are used to aid in computer forensics are:

  • write blocker: used as a one way access to read data from disk and prevent any data from being written on it.
  • FTK imager: used to create an image of digital evidence for analysis, this can even be used to perform search on the drive.
  • RegMon: this is used to monitor registory activity
  • Reg-Ripper: this is used to locate and access registry key nodes and data. translates data into readable form.
  • Photorec: this is used to recover data.
  • Volatility: this used for memory forensics.

Analysis

The analysis of the data can be done using the tools mentioned before. Using reg-ripper we can easily locate the registry data and using the tool to even translate data into readable strings. Using photorec, we can recover data that are deleted or not shown easily. Using volatility we can examine memory that we have dumped and see processes that were stored or run. Different tools can help us reveal a lot of information in the system, for example, we can easily search for passwords that are stored.

All of these tools are all used to perform similar functions which is to look for printable string, reconstruct the data structure in the memory, and search for static signatures or kernel data. In addition to memory analysis, we are also interested in registry that are changed or created. Because of the way windows system is designed, it will record and keep registry of what is modified and the time so this can help us identify applications and their value. As we have learned about malwares, they want to survive system reboot so they will create a registry to allow the malware to automatically run when the system is started up and by analyzing the registry we can search for the malware and see the modified times in correlation with other time stamped files to develop a forensics timeline.
