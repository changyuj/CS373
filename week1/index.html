Malware Basics

From the video lecture, I have learned the definition of malware is a malicious software that is used to carry out unwanted behavior to a computer system. When we further analyze it, we can find out there the motives that drives these malware creations can be either destruction, spy, political gain or financial gains.

Through the research of malwares, they can be classified into Viruses, Trojans, and Potentially Unwanted Programs.

  • Virus: These are files that are executable and are usually triggered when we run the program. (ie. running an downloaded infected program).
  • Worm: These are types that will spread itself as fast as possible with no user interaction. (ie. replication of the program in the background).
  • Trojan: These types will usually be hidden inside known file types that the user are familiar with (ie. word doc, excel, PDF… etc). These programs will find a vulnerability in your system and install a backdoor that will allow the intruder to gain access to your system at anytime.
  • PUP, PUA, PUS: unwanted programs includes adwares, spywares and some harmful or intruding tools

How malwares work

In order for anyone to protect themselves against any malicious software, we would need to understand how the malwares work. The easiest way to understand how the malwares work is by tracing back to how the system get infected in the first place and understand what vectors are used to infect the system. Common vectors are file types that users are familiar with so they will drop their guard and open up the file like the office files, PDF and emails. Other vector can be from unofficial downloads of apps and programs that user try to get for free and the apps and program can be altered without our knowledge. Another vector can be the use of USB sticks that does not belong to you and maybe tempered with. Common vector is internet, where user goes to unsecure or unsafe websites that are compromised. By understanding how these malwares are spread now gives us a fighting chance to protecting ourselves thanks to the antivirus research.

Malware research

Thanks to top researchers and antivirus companies out there that are working around the clock studying and developing ways to crack the malwares and help keeping our system clean. They are like the doctors who prescribe medicines when we are sick and give us vaccination shots to prevent us from getting sick if the virus is seen again.

Malware researchers obtain the infected files and call them samples and keep them contained in an ‘inactive’ state so they are safe to be transported to other systems for studying. To study the samples, they would either study the code (static analysis) or have very strict environment of systems that are under their control and watch how the malware works(dynamic analysis>. Understanding what its purpose is a key to defeating it. Researchers will have to use multiple analytic tools to help them dissect the malware and find out where it is going, what it is hiding and what did it do. Some of these tools are provided to us to use in our labs like process explorer, process monitor, flypaper, and antispy…etc.

In addition, the malware threats are classified into three major categories.

  • Advanced – These attacker are professionals that can make their own exploits and experienced in intrusion of systems.
  • APersistent – these attackers have an target that they will work to get that target reached undetected.
  • Threat – these attacker are the most dangerous since they are organized and will funded and are motivated by some purpose.

Week 1 Lab

In this week, we have a lab that get us a taste of how malware researchers study and analyze the malware (dynamic analysis). First we are given access to virtual machines that are isolated and before we start anything, always take a snapshot of the system because this is where you can always go back to when you lose control of the malware or you need to exam the action of the malware again from the beginning. The before mentioned tools are provided to us and we must run them first before running the malware sample. We used FlyPaper to stop program from exiting and block network. FakeNet, as the name suggests, set up a fake network protocol that will capture network requests. ProcessMonitor that shows us a log of timestamped program execution steps in the system. This is very useful to trace the program to see what it has done. ProcessExplorer is very similar to task manager that will show us the process that are executed on the system. AntiSpy provided many great features for us to investigate system files, system registry and properties. FileInsight is a hex editor that is great to look at the binary code of the malware that can help use decrypt the file. Another great tool to use is to calculate the MD5 hash or the SHA1 hash of the target file and to make sure a file is official or has it been tempered with by checking it against virustotal's file bank.
